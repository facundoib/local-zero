# v0.1 Model Selection — Resolution

**Date:** 2026-05-06 (initial); re-opened and re-resolved 2026-05-08.
**Status:** Resolved. SPEC §5.3 chat + embedding defaults locked. **Chat default swapped from `Qwen3-14B-GGUF` to `Qwen3-4B-Instruct-2507-GGUF` on 2026-05-08 — see "2026-05-08 re-open" section below.**

This document records the decision to **hold** the SPEC §5.3 defaults
against the wave of 2026-released candidate models (Qwen3.5 / Qwen3.6,
Gemma 4, Magistral-Small-2509). Conservative threshold applied: a swap
requires (a) Lemonade-registered as a managed recipe, (b) measurable
Spanish-RAG quality win, (c) no open llama.cpp Vulkan stability bugs
against the candidate. As of 2026-05-06, no 2026-released candidate
satisfies all three.

Full evidence: [`deep research/compass_artifact_wf-63a144ba-…_text_markdown.md`](../../../deep%20research/compass_artifact_wf-63a144ba-5840-4324-82ae-7cf1a7e01f70_text_markdown.md) (workspace-level, outside the repo). Compressed below.

## Operational mitigation: Qwen3-family code-switching

The Qwen3 dense models are documented to "think in English" by default
even with non-English system prompts (arXiv 2508.10355 "Making Qwen3
Think in Korean with RL", August 2025; QwenLM/Qwen3.5#35
"Qwen3.5-series models always think in english", 2026-02-27). The Qwen3
official model card warns: *"using a higher [presence_penalty] value may
occasionally result in language mixing"* (huggingface.co/Qwen/Qwen3-14B
Best Practices section).

Two mitigations are now in the codebase:

1. **System prompt hardening** — both [`src/Chat.tsx`](../../src/Chat.tsx) and the locked SPEC §F5 prompt include:
   - *"Respondé SIEMPRE en español, sin alternar al inglés bajo ninguna circunstancia."*
   - *"No 'pienses' en inglés y traduzcas: razoná directamente en español."*

2. **`enable_thinking: false`** sent on every `/v1/chat/completions`
   request via `chat_template_kwargs` — the Qwen3 registry recipe is the
   reasoning variant, and default-on thinking traces are the documented
   carrier of the English bias. See [`src/lemonade.ts`](../../src/lemonade.ts) `chatStream`.

3. **`presence_penalty` left at API default 0.0.** SPEC §F5 now records
   this as a locked parameter with the rationale (raising above ~0.5
   triggers language mixing per the Qwen3 model card).

## Why no swap to 2026-released models

| Candidate | Disqualifier (single sentence) |
|---|---|
| Qwen3.5-9B / -4B | Lemonade registry support is in flight (PR #1290) but the family inherits the same English-thinking defect (QwenLM/Qwen3.5#35) and Lemonade has an open `enable_thinking=false` bug (Lemonade #1511, opened 2026-04-02). |
| Qwen3.5-27B dense | ~17.5 GB at Q4_K_M — does not fit 16 GB target. |
| Qwen3.6-35B-A3B (MoE) | ~17 GB total weights — does not fit 16 GB at full GPU offload; CPU offload destroys interactive latency. |
| Gemma-4-26B-A4B (MoE) | Three open llama.cpp Vulkan stability bugs (#21516 infinite `<unused8>` tokens, #21400 invalid-pointer crash, #21690 ctx-checkpoint OOM); plus ~17 GB total. |
| Gemma-4-E4B (dense) | Same Gemma-4 Vulkan issue cluster — risky on RDNA4 target until those close. |
| Gemma-3-12B-it | Fits comfortably and is mature in llama.cpp, but has no published Spanish RAG quality measurement that justifies displacing Qwen3-14B's documented 119-language pretraining. |
| Magistral-Small-2509 | At Q4_K_M (14.3 GB) it is borderline OOM at 8K context; chat template requires `mistral-common` sidecar service (per the model card: *"in most likelihood incorrect"* with the embedded template). Non-trivial integration cost. |
| Llama-3.1-8B-Instruct | The **only** candidate with a published La Leaderboard ES score (59.03) and is Lemonade-registered, but only 8 supported languages and an older training cutoff make it a regression on capability vs Qwen3-14B's 119-language pretraining. Tracked as a re-evaluation tier. |
| Llama 4 (Scout / Maverick) | Total weights >50 GB at Q4 — out of 16 GB budget by an order of magnitude. |

## Re-open thresholds

This decision **re-opens** if any of the following holds. Until then,
treat it as locked.

1. A 2026-released, Lemonade-registered chat model posts a **≥3-point ES delta** on La Leaderboard (or an internal Argentine-Spanish PDF QA harness) over Qwen3-14B, AND has zero open llama.cpp Vulkan stability bugs.
2. A community fine-tune of Qwen3.5-9B demonstrably eliminates code-switching on a Spanish system prompt (the analog of "Smoothie Qwen3" for Chinese), AND the Lemonade `enable_thinking=false` bug (#1511) is closed.
3. Lemonade adds **BGE-M3 to `server_models.json` as an embedding** (not just the reranker that's there today), AND an internal nDCG@10 measurement on Spanish PDFs shows BGE-M3 beats Qwen3-Embedding-0.6B by ≥3 points.
4. The llama.cpp Gemma-4 Vulkan issues (#21516, #21400, #21690) all close, AND a published Spanish QA evaluation shows Gemma-4-E4B beats Qwen3-14B on a Spanish RAG harness.

## Follow-up work

- **Internal Spanish-RAG harness** — 50-100 Argentine-Spanish PDF QA pairs. Without it, every future swap proposal flies blind on the actual user-visible quality. Tracked as a v0.1 release-blocker for any model swap, not for the v0.1 ship itself.
- **`Qwen3-Embedding-0.6B-GGUF` registry visibility** — the 0.6B variant was not directly visible in `server_models.json` at inspection (only the 8B). Lemonade auto-pulls on first `/v1/embeddings` call (verified 2026-04-30, [OQ#1](v0.1-open-questions.md#oq1)), so this is not a release blocker, but worth confirming with Lemonade maintainers before SPEC §5.3 finalizes.
- **Vulkan as default backend on RDNA4** — the deep research finds Vulkan beats vLLM ROCm on RX 9070 XT for current llama.cpp builds (62 vs 48 tok/s on Qwen3.5-9B Q6_K, ivangotoy/llama-server-rdna4-vulkan). Not a SPEC change today since Lemonade picks the backend, but a note for the README / acceptance docs.

## 2026-05-08 re-open — Qwen3-14B-GGUF demoted to opt-in upgrade

**Trigger:** F5 + RAG splice acceptance test (SPEC §12.3 step 3) failed against `Qwen3-14B-GGUF`. Output collapsed into degenerate token streams (1-2 leading tokens followed by hundreds of `?` characters) on every prompt, RAG or non-RAG.

### Evidence

All tests against the same Lemonade v10.3.0 install on a single dev machine (RTX 5060 Ti 16 GB, Windows 11). All requests sent `chat_template_kwargs:{enable_thinking:false}` per SPEC §F5. `temperature=0.4`, `stream=false` for the diagnostic captures.

| Test | Prompt | Model | Result |
|---|---|---|---|
| T1 | `¿Cuánto es 2 más 2?` (no system, no RAG, ~10 tokens in) | Qwen3-14B-GGUF | `并于???????????` (2.1 s, ~80 tokens out, mostly `?`) |
| T1b | `qué hora del día son las 15:00` (no system, no RAG) | Qwen3-14B-GGUF | `uno??????????` (1.9 s) |
| T2 | Clean Spanish RAG-style prompt: 2 fragments of fabricated bicycle-manual content (no `�`, no lorem ipsum, no repetition), `¿Cuánto pesa la Cumbre 29?` | Qwen3-14B-GGUF | `forth???????????????` (4.7 s) |
| F5 splice via WebView | Real PDF chunks (encoding-corrupted by pdf-extract, lorem ipsum repetition), `¿Quién es María?` | Qwen3-14B-GGUF | `读书??????????????????` |
| Sanity | Identical RAG prompt to F5 splice | **Qwen3-4B-Instruct-2507-GGUF** | `Mar�a vive en Espa�a y se dedica a la ingenier�a de software.` ✓ correct citation, 19.1 s |

The pattern reproduces across short/long prompts, with/without system prompt, with/without RAG context, with clean inputs and with corrupted inputs. The corrupted PDF fixture (U+FFFD chars from a `pdf-extract` encoding bug, tracked separately) was the original suspect but T1 + T2 rule it out: 14B fails on inputs with no corruption at all.

### Root cause hypotheses (not investigated; cost > value at v0.1)

1. Lemonade v10.3.0's reasoning-template handling for `Qwen3-14B-GGUF` does not propagate `enable_thinking:false` to the underlying llama.cpp chat template, leaving the model in a degenerate decoding mode.
2. The shipped GGUF for `Qwen3-14B-GGUF` (Lemonade-managed download) is corrupt or quantized inconsistently with its tokenizer.
3. A llama.cpp build incompatibility on this Windows / WebView2 stack vs the chat template metadata in the GGUF.

Any of (1)-(3) would be a Lemonade- or llama.cpp-side bug, not a Local Zero bug. Investigating costs more than swapping defaults; we do neither beyond filing observations.

### Decision

`PREFERRED_CHAT_MODELS` in [`src/lemonade.ts`](../../src/lemonade.ts) is reordered to:

```ts
const PREFERRED_CHAT_MODELS = ["Qwen3-4B-Instruct-2507-GGUF", "Qwen3-14B-GGUF"];
```

`Qwen3-4B-Instruct-2507-GGUF` becomes the v0.1 default chat model. The 14B stays in the list as a second preference so a future Lemonade or llama.cpp fix to the reasoning-template path makes the upgrade a no-code-change `lemonade pull Qwen3-14B-GGUF`. SPEC §5.3 and §F5 are updated to match.

### Trade-off accepted

- Capability regression vs the original 14B-locked plan: the 4B is a smaller dense Instruct model. Argentine-Spanish PDF QA quality is below what 14B could deliver in principle. v0.1 ship criterion (§12.3 step 3 — "streams a Spanish-language answer that cites information from the PDF") is satisfied empirically by the 4B; finer quality is deferred to the internal Spanish-RAG harness follow-up.
- The [original 2026-05-06 decision](#decision) compared 14B to alternative *2026-released* models. None of those alternatives become viable today because the 14B's failure mode here is a Lemonade integration / template propagation issue, not a 14B-the-weights issue. Swapping to a different base architecture would inherit the same Lemonade-backend risk.

### Re-open thresholds for restoring 14B as default

This decision **re-opens and restores 14B as default** if any of the following holds:

1. Lemonade ships a release with `enable_thinking=false` propagation verified end-to-end on `Qwen3-14B-GGUF` (manifested as: identical T1/T1b/T2 captures yielding coherent Spanish output instead of degenerate `?` streams), AND a re-run against the internal Spanish-RAG harness (when it exists) shows 14B beats 4B-Instruct on the harness.
2. A non-reasoning Qwen3-14B variant is added to the Lemonade registry (e.g., a hypothetical `Qwen3-14B-Instruct-2507-GGUF` recipe), AND fits the same VRAM / quality criteria.
